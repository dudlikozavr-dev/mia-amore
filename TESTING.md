## Mia-Amore Telegram Mini App

---

## 1. Что это за проект

Telegram Mini App — интернет-магазин женской домашней одежды.  
Работает **только внутри Telegram** (полный функционал) или в браузере (без Telegram API).

**Технологии:** чистый HTML + CSS + ванильный JS, без фреймворков.  
**Хостинг:** Vercel — `https://tg-app-red-three.vercel.app`  
**Прямая ссылка на бота:** `https://t.me/sikretsweet_home_bot/app`  
**GitHub:** `https://github.com/dudlikozavr-dev/mia-amore`

---

## 2. Как запустить локально

### Вариант A — Node.js (рекомендуется)

```bash
cd tg-app
npx serve .
# Откроется на http://localhost:3000
```

### Вариант B — Python

```bash
cd tg-app
python -m http.server 8080
# Откроется на http://localhost:8080
```

> **Важно:** открывать именно папку `tg-app/`, а не корень проекта.  
> Двойной клик на `index.html` не работает — браузер блокирует модули по `file://`.

### Вариант C — Vercel (прод)

```bash
npm i -g vercel
cd tg-app
vercel --prod
```

## 7. Бизнес-логика доставки и скидок

**Доставка** (из `store.js → getDelivery()`):

```
Сумма заказа >= 3 000 ₽  →  Бесплатно
СДЭК (cdek)              →  300 ₽
Почта России (post)      →  250 ₽
```

**Скидка:**

```
mia_discount = "0.1"  →  скидка 10% от суммы товаров
getDiscountAmount()    →  Math.round(subtotal * 0.1)
```

Скидка применяется в корзине и оформлении. **Не влияет на итог на экране успеха** (это известный баг — см. раздел 11).

## 8. Навигация — карта переходов

```
[Онбординг] → (кнопка «Смотреть каталог») → [Оффер] → [Каталог]

Нижняя навигация (таб-бар):
  Главная ↔ Избранное ↔ Корзина ↔ Профиль

[Каталог] → (клик на карточку) → [Товар]
[Избранное] → (клик на карточку) → [Товар]
[Товар] → (Telegram BackButton) → предыдущий экран

[Корзина] → (MainButton «Оформить заказ») → [Оформление]
[Оформление] → (BackButton) → [Корзина]
[Оформление] → (MainButton «Оплатить») → [Подтверждение]
[Подтверждение] → (кнопка «Вернуться в каталог») → [Каталог] + reset()
```

**Экраны с нижним таб-баром:** Каталог, Избранное, Корзина, Профиль.  
**Экраны без таб-бара:** Товар, Оформление, Подтверждение.

### TC-08 Корзина

**Предусловие:** добавить хотя бы 1 товар.

| Шаг | Действие | Ожидаемый результат |
|---|---|---|
| 1 | Открыть пустую корзину | «Корзина пуста» + кнопка «Перейти в каталог» |
| 2 | Открыть корзину с товарами | Список товаров + блок «Итого» снизу |
| 3 | Нажать «+» | Количество увеличивается, цена пересчитывается |
| 4 | Нажать «−» при qty=1 | Диалог подтверждения «Удалить товар из корзины?» |
| 5 | Подтвердить удаление | Товар удалён, список обновлён |
| 6 | Отменить удаление | Товар остаётся |
| 7 | Нажать ✕ (крестик) | Диалог подтверждения, аналогично |
| 8 | Сумма < 3 000 ₽ | Подсказка «Добавьте ещё на X ₽ — доставка бесплатно» |
| 9 | Сумма >= 3 000 ₽ | Подсказка пропадает, доставка = «Бесплатно» |
| 10 | Скидка есть (mia_discount=0.1) | Строка «Скидка 10% −X ₽» видна зелёным |
| 11 | Скидки нет | Строка скидки скрыта |
| 12 | В Telegram: MainButton | «Оформить заказ — X ₽» (сумма с учётом скидки + доставки) |

**Расчёт: добавь «Bella» (8 990 ₽):**
- Без скидки, СДЭК: 8 990 + 300 = **9 290 ₽** (но доставка бесплатна при > 3000, значит 8 990 + 0 = **8 990 ₽**)
- Со скидкой 10%: скидка = round(8990 × 0.1) = **899 ₽**, итого = 8 990 − 899 + 0 = **8 091 ₽**

## 11. Известные ограничения и баги

### BUG-01 — Итоговая сумма на экране успеха не учитывает скидку

**Файл:** `js/success.js`, строка 44  
**Код:** `const total = order.subtotal + order.delivery;`  
**Проблема:** скидка (`order.discount`) не передаётся в объект `order` и не вычитается.  
**Воспроизведение:** применить скидку → оформить заказ → итог на экране успеха выше, чем было в корзине.  
**Приоритет:** Medium

---

### BUG-02 — CloudStorage корзины не работает в браузере

**Ожидание:** корзина сохраняется между сессиями.  
**Факт:** `window.Telegram?.WebApp?.CloudStorage` = `undefined` в браузере, `try/catch` глотает ошибку.  
**Воспроизведение:** добавить товар → закрыть вкладку → открыть снова → корзина пуста.  
**Статус:** ожидаемое поведение вне Telegram, не баг.

---

### Известные ограничения (не баги)

- Нет реального бэкенда: заказ «оформляется» с задержкой 1 секунда (`setTimeout(r, 1000)`) и не отправляется никуда.
- Фотографии товаров — SVG-заглушки, сгенерированные в data.js.
- `Telegram Pay` на экране оплаты — заглушка, реальная оплата не подключена.
- Кнопка «Написать в чат» в таблице размеров вызывает `TG.close()` — закрывает Mini App.
- Номер заказа генерируется случайно: `Math.floor(1000 + Math.random() * 9000)` — 4 цифры.
