## 1. КОНКУРЕНТЫ И ПРИМЕРЫ

### Проанализированные бренды (10 лендингов)

| Бренд | Стиль | Что цепляет |
|---|---|---|
| **Coco de Mer** | Dark romantic, золото | Тёмный фон + золотые акценты; мощная героиня, эстетика роскоши |
| **Deshabille** | Dark luxury | Близок к нашей палитре `#0d0a0f` + оранжевый/золото |
| **Olivia von Halle** | Эдиториал, белый | Огромные фото на модели, минималистичная сетка |
| **Lilysilk** | Информационный | Блок «Почему шёлк?» — объяснение материала обязательно |
| **Lunya** | Лайфстайл, светлый | Продаёт образ жизни, а не товар; «sleep better» |
| **Silk Maison** | Чистый, editorial | Hover-эффект: фото на вешалке → фото на теле |
| **Homebodii** | Уютный, пастельный | Сильный блок «подарок», упаковка как аргумент |
| **Rya Collection** | Минимализм | Видео-петля в героине, текстура ткани крупным планом |
| **Hanami Dream** | Японская эстетика | Сакура, иероглифы как декор, мягкие градиенты |
| **Anthropologie** | Lifestyle editorial | Каждый цвет — отдельная история, lifestyle-съёмка |

### Главные выводы о конкурентах

1. **Фото на теле > фото товара.** Hover-эффект «вешалка → модель» — стандарт. Снимает возражение «как это будет смотреться на мне».
2. **Объяснение материала — обязательный блок.** Покупательница должна понять, за что платит, до CTA.
3. **Подарочная тематика удваивает аудиторию.** Блок «Подари себе или близкой» + красивая упаковка — часто половина трафика перед праздниками.
4. **Лучший ориентир для нас:** Coco de Mer и Deshabille — dark romantic с золотыми акцентами идеально ложится на нашу палитру.

## 2. СТРУКТУРА ПРОДАЮЩЕГО ЛЕНДИНГА

### Идеальные 9 блоков (порядок важен)

#### Блок 1 — HERO (захват)
- **Что:** Полноэкранный заголовок + фоновый образ/градиент + одна CTA-кнопка
- **Цель:** За 3 секунды ответить «что это, для кого и зачем»
- **Критично:** H1 должен описывать результат, не товар. «Твоё утро начинается здесь», а не «Халаты кимоно».

#### Блок 2 — ДОВЕРИЕ / СТРОКА ГАРАНТИЙ
- **Что:** 3–4 иконки с подписями: доставка, оплата при получении, возврат, время ответа
- **Цель:** Снять тревогу ДО прокрутки к товарам
- **Критично для РФ:** Оплата при получении — приоритет #1 для первой покупки

#### Блок 3 — ПРОДУКТОВАЯ ГАЛЕРЕЯ
- **Что:** Карточки товаров: фото + название + цена + CTA
- **Цель:** Визуальный выбор — главный аргумент в fashion
- **Фишка:** Hover — альтернативное фото на модели

#### Блок 4 — ПОЧЕМУ ИМЕННО ЭТОТ МАТЕРИАЛ
- **Что:** 3–4 тезиса о ткани с иконками (мягкость, гигроскопичность, уход)
- **Цель:** Обосновать цену, снять возражение «дорого»
- **Формат:** Продавать ощущение, не характеристику: «Такой мягкий, что хочется не вставать» вместо «280 г/м²»

#### Блок 5 — ОТЗЫВЫ
- **Что:** 3–6 отзывов с именем, фото (если есть), оценкой
- **Цель:** Социальное доказательство — самый недооцененный блок
- **Критично:** Без отзывов конверсия падает на 30–40% у незнакомых брендов

#### Блок 6 — ТАБЛИЦА РАЗМЕРОВ
- **Что:** Визуальная таблица ОГ/ОТ/рост → размер
- **Цель:** 47% покупателей проверяют размер перед заказом одежды онлайн
- **Фишка:** Кнопка «Не знаю свой размер — спроси Дениса» → Telegram

#### Блок 7 — ОФФЕР / ЦЕНА / УСЛОВИЯ
- **Что:** Финальная цена, что входит (халат + упаковка?), сроки доставки, CTA
- **Цель:** Закрытие возражений + импульс к действию
- **Фишка:** Счётчик остатка «Осталось 3 штуки в этом цвете»

#### Блок 8 — ПОДАРОЧНЫЙ СЦЕНАРИЙ
- **Что:** «Это тот подарок, которым пользуются каждый день»; фото упаковки
- **Цель:** Привлечь тех, кто ищет подарок (мама, подруга, себе любимой)
- **Формулировка:** «Выбрать в подарок» как отдельный CTA

#### Блок 9 — ЛИЧНОСТЬ ПРОДАВЦА / КОНТАКТ
- **Что:** Фото Дениса, 2–3 предложения «кто я», ссылка на Telegram
- **Цель:** Живой человек vs. безликий магазин — огромное конкурентное преимущество
- **CTA:** «Написать Денису в Telegram» (ссылка через `tg://`)

## 3. ТРЕНДЫ ВЕБ-ДИЗАЙНА 2025–2026

### 15 трендов с возможностью реализации без библиотек

#### Визуальные тренды

**Тренд 1 — Glassmorphism 2.0**
- Дисциплинированное размытие: `backdrop-filter: blur(20px)` только на карточках
- Иридесцентные границы: тонкая линия с градиентом оранжевый → розовый → золотой
- НЕ размывать весь фон — только отдельные элементы

**Тренд 2 — Noise/grain overlay**
- Inline SVG data URI как `::before` псевдоэлемент поверх секций
- Даёт «плёночную» текстуру без внешних файлов
- `opacity: 0.03–0.05` — едва заметно, но добавляет глубину

**Тренд 3 — Ambient halo lighting**
- Дышащая анимация орба: `@keyframes breathe { scale: 1 → 1.15 }`
- Mouse parallax: орб слегка следует за курсором (JS 10 строк)
- Увеличить opacity орбов с текущих 0.12 до 0.18–0.20

**Тренд 4 — @property анимированные CSS-переменные**
- `@property --gradient-angle` → плавная анимация градиентных границ
- Работает в Chrome 85+, Safari 15.4+
- Применение: кнопки CTA, рамки карточек

**Тренд 5 — Grain текстура на градиентах**
- `filter: url(#noise)` поверх hero-градиента
- SVG-фильтр инлайном в HTML — нет внешних зависимостей

#### Анимации и взаимодействие

**Тренд 6 — Scroll-driven animations (нативные)**
```css
@keyframes reveal {
  from { opacity: 0; transform: translateY(30px); }
  to   { opacity: 1; transform: translateY(0); }
}
.card {
  animation: reveal linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 40%;
}
```
Chrome 115+, Safari 18+. Fallback — Intersection Observer.

**Тренд 7 — Intersection Observer reveal (fallback)**
```js
const observer = new IntersectionObserver(entries => {
  entries.forEach(e => e.isIntersecting && e.target.classList.add('visible'));
}, { threshold: 0.15 });
document.querySelectorAll('.reveal').forEach(el => observer.observe(el));
```

**Тренд 8 — Card shine/spotlight на hover**
```js
card.addEventListener('mousemove', e => {
  const rect = card.getBoundingClientRect();
  const x = ((e.clientX - rect.left) / rect.width) * 100;
  const y = ((e.clientY - rect.top) / rect.height) * 100;
  card.style.setProperty('--mx', x + '%');
  card.style.setProperty('--my', y + '%');
});
```
CSS: `background: radial-gradient(circle at var(--mx) var(--my), rgba(255,255,255,0.06), transparent 60%)`

**Тренд 9 — 3D tilt карточек**
```js
card.addEventListener('mousemove', e => {
  const rect = card.getBoundingClientRect();
  const x = (e.clientY - rect.top  - rect.height/2) / 15;
  const y = (e.clientX - rect.left - rect.width/2)  / 15;
  card.style.transform = `perspective(600px) rotateX(${-x}deg) rotateY(${y}deg)`;
});
card.addEventListener('mouseleave', () => card.style.transform = '');
```

**Тренд 10 — Magnetic кнопка CTA**
```js
btn.addEventListener('mousemove', e => {
  const rect = btn.getBoundingClientRect();
  const x = (e.clientX - rect.left - rect.width/2)  * 0.3;
  const y = (e.clientY - rect.top  - rect.height/2) * 0.3;

btn.addEventListener('mouseleave', () => btn.style.transform = '');
```

## 4. UX И ТРИГГЕРЫ КОНВЕРСИИ

### Психологические триггеры (по силе)

1. **Фото реальных покупателей в товаре** — сильнейший триггер в fashion
2. **Дефицит с конкретной цифрой** — «Осталось 3 штуки» работает; «почти закончилось» — нет
3. **Личность продавца** — Денис как живой человек с Telegram уже преимущество перед безликими магазинами
4. **Образ жизни вместо товара** — продавать «утренний ритуал», а не «атласный халат»

### CTA: лучшие формулировки

| Слабо | Сильно |
|---|---|
| «Купить» | «Выбрать свой халат» |
| «Заказать» | «Написать Денису в Telegram» |
| «Подробнее» | «Хочу этот уют» |
| «Добавить в корзину» | «Забрать себе» |
| — | «Выбрать в подарок» |

### Обязательные элементы доверия

- Строка гарантий под hero: доставка / оплата при получении / возврат / время ответа
- Отзывы с именем (фото — если есть)
- Таблица размеров (47% покупателей проверяют перед заказом одежды)
- Sticky CTA на мобильном внизу экрана (+15–20% конверсии)

### Специфика российской аудитории

- **Оплата при получении** — приоритет #1 для первой покупки
- Telegram-ссылка через `tg://` протокол (открывается напрямую, не в браузере)
- Обращение на «ты» снижает психологический барьер
- Явное указание города/региона или «доставка по всей России»

## 5. ЦВЕТА И ТИПОГРАФИКА

### Текущая палитра + рекомендации

```
Текущее:  #0d0a0f (фон) + #f59e0b (акцент) + белый текст
Проблема: белый текст слишком холодный для feminine luxury
Решение:  заменить #ffffff на #f0e8d8 (тёплая слоновая кость)
```

### Расширенная палитра

```css
--bg:        #0d0a0f;   /* основной фон */
--surface:   #1a1420;   /* фон карточек */
--gold:      #d4a843;   /* основной акцент (теплее, чем #f59e0b) */
--rose:      #c9748a;   /* второй акцент */
--cream:     #f0e8d8;   /* текст (вместо чистого белого) */
--blush:     #e8b4b8;   /* нежные детали */
--mauve:     #9c6b7a;   /* приглушённый акцент */
--plum:      #3d1a2e;   /* третий цвет орба */
```

### Типографика (главный рычаг улучшения)

**Текущее:** `-apple-system, BlinkMacSystemFont, 'Segoe UI'` → читается как tech-стартап

**Рекомендация A (роскошная классика):**
```html
<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:ital,wght@0,400;0,500;1,400&family=Jost:wght@300;400;500&display=swap" rel="stylesheet">
```
- Заголовки: `Playfair Display`, `weight 400–500`, italic для тэглайнов
- Тело: `Jost`, `weight 300` (body), `weight 400` (UI)

**Рекомендация Б (японская эстетика):**
```html
<link href="https://fonts.googleapis.com/css2?family=Cormorant+Garamond:ital,wght@0,300;1,300&family=DM+Sans:wght@300;400&display=swap" rel="stylesheet">
```
- Заголовки: `Cormorant Garamond` — ультратонкий каллиграфический serif, отсылка к японской туши
- Тело: `DM Sans` — чистый, современный

#### Counter animation (для секции с числами)

```js
function animateCounter(el, target, duration = 1500) {
  const start = performance.now();
  const update = now => {
    const t = Math.min((now - start) / duration, 1);
    el.textContent = Math.round(t * target);
    if (t < 1) requestAnimationFrame(update);
  };
  requestAnimationFrame(update);
}
```

### Описание материала: как писать

```
Слабо: «Состав: 100% микрофибра, плотность 280 г/м², двойной шов»

Сильно: «Такой мягкий, что хочется завернуться и не вставать.
         Не скользит, не мнётся — просто остаётся на тебе
         в течение всего утра. Даже после стирки.»
```

### Формула описания секции с материалом

```
[Ощущение] → [Причина (материал)] → [Практичный факт]

Пример:
«Мягкий как второе дыхание — благодаря микрофибре с начёсом.
 Не скользит с плеч. Машинная стирка при 40°.»
```

## ЭКСПЕРТНАЯ ОЦЕНКА

### 🎨 Эксперт 1: UX-дизайнер (10 лет опыта)

#### Типичные ошибки новичков в структуре

1. **Hero без ответа на вопрос «что это»**. Красивый градиент и заголовок «Твоё утро» — хорошо, но если нет подзаголовка с пояснением («женские халаты кимоно из микрофибры»), часть посетителей уйдёт, так и не поняв что продаётся.

2. **CTA без контекста**. Кнопка «Купить» в hero до того, как пользователь увидел товар, цену и хоть один отзыв — преждевременное давление. Первый CTA лучше формулировать мягко: «Смотреть коллекцию» или «Выбрать свой».

3. **Нет визуальной иерархии между секциями**. Если все блоки выглядят одинаково — пользователь теряет ориентацию. Нужно чередовать: плотная секция → воздушная, тёмная → чуть светлее.

4. **Мобильные карточки в 3 колонки**. На 360px три карточки рядом — нечитаемо. Максимум 2 колонки на мобильном, лучше 1 с горизонтальным скроллом.

5. **Форма или CTA без объяснения «что будет дальше»**. «Написать в Telegram» — хорошо, но добавь: «Отвечу в течение 10 минут, помогу выбрать размер». Снимает тревогу перед первым контактом.

6. **Отсутствие визуального завершения страницы**. Лендинг должен заканчиваться финальным CTA + контактом. Без этого пользователь «вываливается» в пустоту.

#### Что ОБЯЗАТЕЛЬНО должно быть

| Элемент | Почему критично |
|---|---|
| Чёткий подзаголовок в hero | Пользователь должен за 3 сек понять: что, для кого, почему сейчас |
| Фото товара на человеке | Fashion без примерки — это барьер №1 |
| Цена видна до CTA | Скрытая цена = недоверие |
| Отзывы с именем | Безымянные «★★★★★» не работают |
| Таблица размеров | 47% уходят без покупки если не могут понять свой размер |
| Строка гарантий | Возврат, доставка, оплата — обязательно выше fold |
| Мобильный sticky CTA | Кнопка всегда видна — +15–20% конверсии |

#### Что раздражает и заставляет уйти

- Анимации, которые **мешают читать** (текст появляется слишком медленно, пользователь ждёт)
- **Всплывающие окна** в первые 5 секунд («Подпишись на скидку!»)
- **Мелкий шрифт** на мобильном — меньше 16px для основного текста недопустимо
- **Нет цены** или цена появляется только в Telegram — вызывает подозрение
- **Слишком много текста в hero** — 3+ предложения в подзаголовке убивают внимание
- **Одинаковые CTA везде**: если 5 раз подряд «Купить», это давит, а не убеждает

---

### 📊 Эксперт 2: Маркетолог по конверсии

#### Оффер, который зацепит

Сильный оффер состоит из трёх частей: **результат + конкретика + снятие риска**.

```
Слабо:  «Купи халат кимоно»
Сильно: «Халат кимоно из мягкой микрофибры — 2 490 ₽
         Доставка по России. Оплата при получении.
         Не подошёл — вернём деньги.»
```

**Формула оффера для нашего лендинга:**
> «[Ощущение/результат] за [цена] — [снятие риска]»
> Пример: «Уютное утро каждый день — от 1 990 ₽. Оплата после получения.»

#### Триггеры доверия (в порядке важности)

1. **Реальные отзывы** — имя + текст (+ фото если есть). Самый сильный триггер для незнакомого продавца.
2. **Личность продавца** — Денис с фото и «пишите в Telegram» > безликий магазин.
3. **Оплата при получении** — для российской аудитории это фундамент доверия при первой покупке.
4. **Конкретные цифры** — «Уже 340 заказов» или «Отвечаю за 10 минут» работают лучше обтекаемых «качество гарантируем».
5. **Дефицит** — «Осталось 4 штуки в бежевом» — реальный стимул, но только если правда.
6. **Гарантия возврата** — даже если возвратов почти нет, само наличие снимает тревогу.

#### Путь посетителя: от «зашёл» до «написал»

```
1. Hero (0–3 сек)
   → Понял: «халаты кимоно, тёмный стиль, выглядит дорого»
   → Решение: остаться и посмотреть

2. Строка доверия (3–5 сек)
   → Снял тревогу: «оплата при получении — ок, попробую»

3. Галерея товаров (5–20 сек)
   → Нашёл подходящий цвет/модель
   → Наводит на карточку — видит фото на модели

4. Блок «Почему этот материал» (20–35 сек)
   → Убедился в качестве, понял за что платит

5. Отзывы (35–50 сек)
   → «Реальные люди покупали, им понравилось»

6. Таблица размеров (50–60 сек)
   → Определил свой размер без вопросов

7. Оффер + цена (60–70 сек)
   → Видит финальную цену, условия, CTA
   → Принимает решение

8. CTA «Написать Денису» (70+ сек)
   → Нажимает → открывается Telegram → пишет
```

**Ключевой инсайт:** большинство уходов происходят на шагах 1–2 (не понял) и 6–7 (не нашёл размер или не увидел цену). Эти два момента — приоритет.

#### Дополнительные конверсионные механики

- **Якорная цена**: зачёркнутая «старая» цена рядом с текущей создаёт ощущение выгоды
- **Подарочный сценарий** как отдельный путь: «Ищешь подарок?» → CTA «Выбрать в подарок» → другое приветствие в Telegram
- **Сезонный крючок**: «К 8 марта осталось X дней — успей заказать с доставкой»

---

### 👤 Эксперт 3: Потенциальный клиентка (женщина 28–42 лет, покупает онлайн)

#### Первые 5 секунд: на что смотрит

1. **Картинка** — сразу понятно, красиво ли? Хочу ли я это?
2. **Цена** — не вижу цену в hero = подозрение («наверное дорого»)
3. **Откуда магазин** — Россия? Есть ли доставка в мой город?
4. **Знакомое ли** — отзывы, цифры, реальный человек за магазином?

#### Что заставит остаться

- **Красивые фото халата на модели** — хочется представить себя
- **Понятная цена без сюрпризов** — стоимость + доставка = итого
- **Небольшой ассортимент** — не 200 позиций, а 5–8 хороших. Проще выбрать.
- **Живой язык** — «Такой мягкий, что хочется не вставать» > «высококачественный материал»
- **Размерная таблица с примером** — «Если рост 168, грудь 90 — берёшь М»
- **Ответ на вопрос «а вдруг не подойдёт»** — возврат, обмен, помощь с выбором

#### Что заставит закрыть вкладку

- Нет цены (или «цена по запросу»)
- Нет отзывов — незнакомый продавец без доказательств
- Фото только на вешалке — непонятно как сидит
- Сложная форма заказа вместо Telegram
- Долгая загрузка или всё ползёт на телефоне
- Нарочито «люксовые» формулировки при средней цене — ощущение обмана
- Pop-up с подпиской сразу при заходе

#### Что реально влияет на решение «написать»

> «Я напишу, если: вижу, что за этим стоит живой человек, цена честная,
> размер понятен и есть хоть один отзыв похожей на меня покупательницы.»

---

### СВОДНЫЕ РЕКОМЕНДАЦИИ (все три эксперта)

#### Красные флаги — исправить до запуска

| Проблема | Решение |
|---|---|
| Нет цены в видимой зоне | Показать цену в hero или сразу под галереей |
| Нет отзывов | Добавить 3–5 реальных отзывов с именами |
| Нет таблицы размеров | Блок «Как выбрать размер» с таблицей + CTA в Telegram |
| Нет строки гарантий | Полоса под hero: доставка / оплата при получении / возврат |
| Нет sticky CTA на мобильном | Фиксированная кнопка внизу экрана |
| CTA «Купить» слишком рано | Заменить на «Смотреть коллекцию» в hero |

#### Жёлтые флаги — усилить после запуска

| Идея | Эффект |
|---|---|
| Фото на модели (hover-эффект) | Сильно снижает барьер «как это выглядит на мне» |
| Личность продавца с фото | Доверие к незнакомому бренду |
| Блок «Подарок» с упаковкой | Расширяет аудиторию и поднимает средний чек |
| Якорная цена (зачёркнутая) | Ощущение выгоды без обмана |
| Счётчик остатка | Импульс к действию — но только если правда |
| Сезонные офферы | 8 марта, НГ — плановые пики трафика |

#### Главный вывод

Лендинг халатов кимоно — это **доверительная продажа**. Покупательница не знает тебя. Она рискует деньгами. Задача лендинга — не «продать», а **снять достаточно страхов**, чтобы она решилась написать в Telegram. После этого — продажа делается живым разговором.

Три кита конверсии для этого проекта:
1. **Видимая цена** + оплата при получении
2. **Реальные отзывы** + живая личность продавца
3. **Ясность размера** + «напишите — помогу выбрать»

## Что мы можем сделать лучше конкурентов

### 1. Фотографии — главный рычаг конверсии
Большинство бутиков используют flat lay или фото на вешалке. Нужна lifestyle-съёмка: халат на модели, в интерьере, детали ткани крупным планом. Формат **3:4 portrait** оптимален для Telegram Mini App.

### 2. Таблица размеров прямо в приложении
Причина №1 брошенных корзин в fashion-нише. Bottom sheet с замерами в сантиметрах прямо в карточке товара — без редиректов.

### 3. Навигация "по назначению"
- "Для себя / В подарок"
- "Хлопок / Шёлк / Велюр"  
- "До 2 000 ₽ / До 5 000 ₽"

Такой подход (как у Flowwow) резонирует с реальными намерениями, а не просто категориями.

### 4. Вишлист, работающий на всех устройствах
Привязан к Telegram ID на сервере. Отложила на телефоне — видит на компьютере.

### 5. Уведомления через бота
- "Ваш заказ отправлен — трек-номер: XXXXXXXXX"
- "Товар из вашего вишлиста снова в наличии"
- "Новая коллекция кимоно — посмотреть?"

Это уникальное преимущество Telegram перед обычным интернет-магазином.

### 6. Отзывы с фото покупателей
Небольшая секция внизу карточки: имя, фото, размер, оценка. Реальные фото от покупателей конвертируют лучше профессиональных.

### 7. Быстрый checkout (3 шага максимум)
Данные из Telegram (`requestContact()`) + адрес + оплата. Никаких длинных форм на одной странице.

## Экспертная оценка: Telegram Mini App для магазина домашней одежды

### UX-дизайнер: ошибки, обязательные элементы, раздражители

#### Типичные ошибки новичков в Mini App

1. **Копируют обычный интернет-магазин.** Mini App — это не сайт в рамке. Здесь нет адресной строки, нет привычного «назад» в браузере, нет вкладок. Новички втискивают полноценный сайт в окно Telegram — и получают уродливый iframe. Нужно проектировать как мобильное приложение: один экран = одна задача.

2. **Игнорируют нативные элементы Telegram.** `MainButton` — это кнопка, которую Telegram рисует сам, и у неё самый высокий tap rate в интерфейсе. Новички вместо неё делают кастомную кнопку внизу страницы, которая перекрывается клавиатурой, залезает под safe area и выглядит чужеродно.

3. **Слишком глубокая навигация.** Больше 3 уровней вложенности — пользователь теряется. В Mini App нет breadcrumbs, нет URL для закладок. Правило: Каталог → Товар → Корзина → Оплата. Не больше.

4. **Перегруженная карточка товара.** Пытаются уместить всё: 10 фото, описание на 500 слов, таблицу размеров, отзывы, похожие товары, «с этим покупают». На 5-дюймовом экране это бесконечный скролл. Лучше: 3-4 фото + цена + размер + одна кнопка. Детали — в bottom sheet по нажатию.

5. **Нет обработки пустых состояний.** Пустая корзина — белый экран. Пустой поиск — ничего. Пустой вишлист — тишина. Каждое пустое состояние должно иметь иллюстрацию + CTA: «Корзина пуста — посмотреть каталог».

6. **Размеры как выпадающий список.** `<select>` на мобильном — это лишний тап, открытие системного пикера, поиск нужного варианта, закрытие. Чипы-кнопки (XS / S / M / L / XL) — один тап и готово.

7. **Забывают про состояние загрузки.** Между тапом и результатом — пустота. Пользователь тапает «Добавить в корзину» и ничего не происходит 0.5 сек. Думает — не сработало, тапает снова. Итог: 3 одинаковых товара в корзине. Решение: мгновенный haptic feedback + анимация кнопки.

#### Что ОБЯЗАТЕЛЬНО должно быть

| Элемент | Почему критично |
|---|---|
| `MainButton` Telegram для главного CTA | Нативный вид, всегда виден, не перекрывается клавиатурой |
| Скелетоны загрузки | Без них кажется, что приложение зависло |
| Выбор размера — чипы, не dropdown | Один тап вместо трёх |
| Фото 3:4 на всю ширину экрана | Fashion продаётся глазами |
| Bottom sheet для деталей | Таблица размеров, описание, состав — без ухода с карточки |
| Бейдж на иконке корзины | Пользователь всегда видит, сколько товаров уже добавил |
| Haptic feedback при действиях | Подтверждение: «да, нажатие сработало» |
| Адаптация к dark/light theme | Без неё half пользователей видят белый текст на белом фоне |

#### Что бесит пользователей Mini App

- **Долгая загрузка при открытии.** Больше 2 секунд — 40% закрывают. Решение: максимальный бандл <200 КБ, lazy load изображений, CDN.
- **Кнопка «назад» закрывает приложение вместо возврата.** Если не перехвачен `BackButton` — пользователь теряет корзину и начинает сначала.
- **Нельзя вернуться к каталогу с сохранёнными фильтрами.** Выбрал «Шёлк / Размер M», зашёл в карточку, вернулся — фильтры сброшены. Бесит до удаления.
- **Клавиатура закрывает поле ввода.** При вводе адреса или промокода — поле уезжает под клавиатуру. Нужно слушать `viewportChanged` и перескролливать.
- **Нет подтверждения удаления из корзины.** Случайный свайп — товар исчез. Нужен `WebApp.showConfirm()`.
- **Фото грузятся сверху вниз, «шторкой».** Без `loading="lazy"` + placeholder — визуальный мусор. Лучше: размытый placeholder → чёткое фото.

### Маркетолог: конверсия, удержание, повторные покупки

#### Что заставит купить (а не просто посмотреть)

1. **Ноль трения при первом визите.** Telegram уже знает имя и телефон пользователя. Не проси регистрацию, не проси email, не проси подтвердить номер. Каждое лишнее поле — минус 10-15% конверсии.

2. **Цена видна сразу.** В каталоге, в карточке, в корзине — везде. «Цена по запросу» или «от 1990 ₽» без конкретики — это отказ. Покупательница домашней одежды принимает решение быстро и эмоционально. Если цена спрятана — эмоция уходит.

3. **Оплата при получении — стартовое доверие.** Для первой покупки у незнакомого бренда в Telegram это фундамент. Telegram Payments — для повторных клиентов, когда доверие уже есть.

4. **«Осталось 2 штуки» — работает, если правда.** Дефицит с конкретной цифрой создаёт импульс. Но если каждый товар «последний» — доверие обнуляется.

5. **Подарочный сценарий — скрытый удвоитель аудитории.** Кнопка «Выбрать в подарок» на главной. Отдельный флоу: «Кому дарите?» → красивая упаковка → открытка. Перед 8 марта, днями рождения — до 50% заказов могут быть подарочные.

6. **Отзывы с фото — главный конвертор.** Не анонимные звёзды, а: «Мария, Казань, размер М» + фото в халате. Это снимает главный страх fashion-покупки: «а вдруг на мне будет выглядеть не так».

#### Конверсионные элементы по экранам

| Экран | Конверсионный элемент | Эффект |
|---|---|---|
| Каталог | Бейдж «Хит» / «Новинка» на карточках | Направляет внимание, ускоряет выбор |
| Каталог | Фильтр «По поводу»: Для себя / В подарок / В роддом | Попадание в намерение |
| Карточка | Фото на модели (не на вешалке) | +25-35% конверсии в fashion |
| Карточка | «Только 2 осталось в этом размере» | Импульс к покупке |
| Карточка | Таблица размеров с рекомендацией | Снимает главный барьер |
| Корзина | «Добавьте ещё на 500 ₽ — доставка бесплатно» | Поднимает средний чек |
| Checkout | Автозаполнение из Telegram | Сокращает форму до 1-2 полей |
| После покупки | Бот: «Ваш заказ отправлен» + трек | Доверие и лояльность |

#### Удержание и повторные покупки

- **Бот-уведомления — уникальное оружие Telegram.** Open rate сообщений бота — почти 100% (против 20% email и 5% push). Используй это:
  - «Новая коллекция — 3 новых кимоно» (раз в 2-4 недели)
  - «Товар из вашего вишлиста снова в наличии»
  - «Скидка 15% на повторный заказ — только сегодня»
- **Не спамь.** Больше 2 сообщений в неделю — пользователь мьютит бота навсегда. Лучше реже, но с ценностью.
- **Персонализация:** «Вы смотрели кимоно из шёлка — вот новый цвет» работает лучше массовой рассылки.

#### Главная метрика

Для мини-магазина домашней одежды в Telegram ключевая метрика — **не количество визитов, а % повторных покупок.** Ниша предполагает 2-4 покупки в год (себе + подарки). Если клиентка вернулась — значит, всё работает.

### Telegram-разработчик: ограничения, возможности, подводные камни

#### Что работает отлично

1. **`WebApp.MainButton`** — синяя кнопка внизу экрана, нативная для Telegram. Используй для каждого CTA: «В корзину», «Оформить», «Оплатить». Она всегда видна, не перекрывается клавиатурой, и пользователь ей доверяет.

2. **`WebApp.BackButton`** — перехвати событие `backButtonClicked` на каждом экране. Без этого кнопка «назад» на Android закрывает всё приложение, и пользователь теряет корзину.

3. **`WebApp.HapticFeedback`** — `impactOccurred('light')` при добавлении в корзину, `notificationOccurred('success')` при оплате. Создаёт ощущение нативного приложения.

4. **`WebApp.requestContact()`** — получаешь номер телефона одной кнопкой. Не надо поле ввода + SMS-подтверждение.

5. **`WebApp.CloudStorage`** — до 1 МБ key-value хранилище на стороне Telegram. Идеально для кэша корзины и вишлиста. Работает между сессиями.

6. **`WebApp.openInvoice()`** — оплата через Telegram Payments. Пользователь видит знакомый интерфейс, не вводит данные карты на незнакомом сайте. Доверие максимальное.

7. **Bot-side сообщения** — после покупки бот отправляет чек, статус доставки, трек-номер. Это не push-уведомление, которое можно пропустить, — это сообщение в Telegram, которое точно прочитают.

#### Ограничения и подводные камни

| Ограничение | Влияние | Обходной путь |
|---|---|---|
| **Нет доступа к GPS** | Нельзя определить город автоматически | Поле «Ваш город» + сохранить в CloudStorage |
| **`<input type="date">` отличается на iOS и Android** | Календарь выглядит по-разному, иногда ломается | Кастомный date picker на JS |
| **Viewport меняется при клавиатуре** | Поля ввода уезжают под клавиатуру | Слушать `viewportChanged`, скролить к активному полю |
| **Нет доступа к камере (кроме QR)** | Нельзя сделать «примерку» через камеру | Обойтись без AR-примерки |
| **CloudStorage — 1 МБ лимит** | Не хватит для полного каталога | Хранить только корзину/вишлист, каталог — на сервере |
| **Telegram Payments недоступны в некоторых странах** | Часть аудитории не сможет оплатить | Альтернатива: ЮKassa / оплата при получении |
| **Нет системы push-уведомлений** | Нельзя отправить push вне бота | Всё через бот-сообщения (которые лучше push) |
| **`initData` протухает** | Сессия не бесконечна | Валидировать на сервере, создавать серверную сессию |
| **Desktop Telegram — другой viewport** | Горизонтальный экран, больше места | Адаптивная верстка: 2-3 колонки на десктопе |

#### Что лучше НЕ делать

- **Не делай горизонтальный свайп для навигации между экранами.** Telegram на iOS перехватывает свайп вправо как «назад». Твой жест и системный жест будут конфликтовать.
- **Не используй `position: fixed` для нижней навигации без `padding-bottom`.** Safe area на iPhone вырезает контент снизу. Используй `env(safe-area-inset-bottom)`.
- **Не загружай весь каталог при старте.** 50+ карточек с фото при открытии = 5+ секунд загрузки. Загружай первые 10, остальные — по скроллу (infinite scroll).
- **Не игнорируй `colorScheme` и `themeParams`.** Половина пользователей Telegram сидит в тёмной теме. Если твоё приложение всегда светлое — оно выглядит как яркая вспышка ночью.
- **Не используй iframe внутри Mini App.** Вложенный iframe в WebView внутри Telegram — это три уровня песочницы. Производительность и совместимость будут ужасными.
- **Не полагайся только на Telegram Payments.** В РФ часть провайдеров работают нестабильно. Всегда имей fallback: ЮKassa, СБП, оплата при получении.

#### Оптимальный технический стек для мини-магазина

```
Frontend:  React/Vue + Vite (бандл <150 КБ gzip)
Стили:     CSS variables из themeParams Telegram
Роутинг:   Простой стейт-машина (не react-router — слишком тяжёлый)
Фото:      WebP, lazy load, srcset для разных экранов
Бэкенд:    Node.js / Python + база для товаров, заказов, пользователей
Хранение:  CloudStorage (корзина) + сервер (заказы, вишлист)
Платежи:   Telegram Payments + ЮKassa + оплата при получении
Бот:       python-telegram-bot или grammY для уведомлений
```
